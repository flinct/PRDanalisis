# Main Impact — prod-2.5.0

## 1. Analytics & Reporting — RBAC & Data Integrity

- Dashboard analytics now enforce self-filtered views based on role (`assigned_only`) to prevent agents from viewing other members' data.
- Offline Report improvements:
  - RBAC visibility restriction applied.
  - Deleted agents are no longer included in report requests.
- Export Ticket Image now uses secure token-based public links instead of direct S3 URLs, with revoke support.

---

## 2. Conversation Experience — Tracking & Consistency

- Added **Conversation Event Logs** to provide a complete timeline for:
  - Assignment changes
  - Status updates
  - SLA changes
  - Sender number changes
  - Team inbox changes
- Fixed counter mismatch between UI and API in:
  - Your Inbox
  - Starred tab
  - Spam tab
- Added notification **Mark as Read** functionality:
  - Per item
  - Mark all as read
  - Direct redirect to conversation/ticket

---

## 3. Mobile App — Outbound Chat Fixes

- Fixed auto-assignment when sending messages from unassigned chats.
- Fixed issue where the first message was not sent from unassigned chat on mobile.

---

## 4. Ticket Management — Workflow Improvements

- Auto-fill assignee with logged-in user during ticket creation (editable).
- Ticket closure restriction:
  - Only creator or supervisor can close tickets.
- Improved responsive layout for ticket pages on smaller viewports.

---

## 5. Contact & Leads — UX Improvements

- Fixed conversation history scrolling in contact detail page.
- Added direct redirect to conversation room from contact detail.
- Fixed non-scrollable channel dropdown in Add Contact form.
- Fixed Leads contact filter issue caused by limit parameter.
